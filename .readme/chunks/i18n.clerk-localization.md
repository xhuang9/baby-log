# Clerk Authentication Localization

> Status: active
> Last updated: 2026-01-17
> Owner: Core

## Purpose

Keep Clerk authentication UI aligned with the app locale and locale-prefixed URLs.

## Key Deviations from Standard

- **English-only base**: `ClerkLocalizations` is configured for `en` only, with fallback to `defaultLocale`.
- **Locale-aware auth URLs**: Auth layout prefixes sign-in/up and redirect URLs when locale is not default.

## Architecture / Implementation

### Components
- `src/config/app.ts` - Defines `AppConfig` and `ClerkLocalizations` mappings.
- `src/app/[locale]/(auth)/layout.tsx` - Picks `clerkLocale` and generates localized auth URLs.

### Data Flow
1. `ClerkLocalizations.supportedLocales` maps locale codes to Clerk resources.
2. Auth layout selects a Clerk locale for the current request.
3. URLs (sign-in/up/bootstrap/sign-out) are prefixed for non-default locales.

### Code Pattern
```ts
const clerkLocale = ClerkLocalizations.supportedLocales[locale]
  ?? ClerkLocalizations.defaultLocale;

if (locale !== routing.defaultLocale) {
  signInUrl = `/${locale}${signInUrl}`;
  signUpUrl = `/${locale}${signUpUrl}`;
}
```

## Configuration

| Setting | Default | Description |
|---------|---------|-------------|
| `ClerkLocalizations.supportedLocales.en` | `enUS` | Clerk locale bundle for English.
| `ClerkLocalizations.defaultLocale` | `enUS` | Fallback locale when unsupported.
| `signInFallbackRedirectUrl` | `/account/bootstrap` | Redirect after sign-in to bootstrap.
| `signUpFallbackRedirectUrl` | `/account/bootstrap` | Redirect after sign-up to bootstrap.

## Gotchas / Constraints

- **Locale coverage**: Adding a new locale requires adding a Clerk localization entry.
- **URL prefixing**: Non-default locales must prefix all Clerk URLs to match app routing.

## Testing Notes

- Switch locale and verify Clerk UI language and URL prefixes.
- Confirm sign-in/up redirect to `/account/bootstrap` with locale prefix when needed.
